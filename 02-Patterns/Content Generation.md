# Content Generation Patterns

1つの素材から複数のコンテンツを一定品質で安定生成するためのパターン集。

---

## 核心原則

**プラットフォームが変わればコンテンツも変わる。同じ文章を流用しない。**

```
X → 短文・断定・速報
LinkedIn → 実績・知見・詳細
Facebook → 信頼形成・長文・ストーリー
note → 深い記事・詳細解説
Threads → 共感・感情・体験談
```

---

## パターン1: 素材→展開（Cascade）

1つの素材を起点にプラットフォーム別に展開する。

```
anchor asset（ブログ記事/議事録/調査結果）
  ↓
3-7つの「atomic idea」を抽出
  ↓
プラットフォーム別に展開
  ↓
重複排除・CTA調整
```

実装例:
```bash
claude -p "
以下の素材から3-7つの核心的な主張を抽出して、
各主張をXとFacebook向けに別々に展開してください。
素材: $(cat /path/to/source.md)
"
```

## パターン2: 声を捕まえる（Voice Capture）

特定の人物の文体・トーンでコンテンツを生成する。

### 声の分析に必要なもの
- 過去の投稿・記事（3件以上）
- 使う言葉の傾向（断定か、問いかけか）
- 文章の長さのリズム
- よく使う構造（事実→解釈 or 問い→答え）
- 避けるべき表現

### 実装
```markdown
【声のサンプル】
[過去投稿を3-5件貼る]

【生成指示】
上記の文体・トーンを完全に模倣して、以下のテーマで[プラットフォーム]向けの投稿を生成してください。
テーマ: [テーマ]
禁止表現: [禁止リスト]
```

## パターン3: 3役分離（Planner/Generator/Evaluator）

品質を保証したい場合（外部公開コンテンツ・売上直結）に使う。

```
Planner (Opus)
  → テーマ決定・伏せる情報の明示・CTA方向
  → /tmp/plan.md に出力

Generator (Sonnet)
  → plan.md に従って本文生成
  → drafts/ に保存

Evaluator (Opus or Sonnet)
  → rubricで採点（7.0未満は差し戻し）
  → approved/ に移動
```

### rubricの設計
- 評価軸を5-7個設定する
- 各軸に重みを付ける（信頼性 × 3 など）
- スコアは10点満点で採点
- 合格ラインを決める（7.0以上など）
- 安全チェックは別途（機密情報・誇張・虚偽）

## パターン4: 品質ゲート

生成後の検証ステップ。Evaluatorに渡す前に自動チェックできるもの。

```bash
# 文字数チェック（Facebookは800文字以上）
WORD_COUNT=$(wc -m < "$DRAFT_FILE")
[ "$WORD_COUNT" -lt 800 ] && echo "文字数不足: $WORD_COUNT 文字" && exit 1

# 禁止ワードチェック
BANNED_WORDS=("クライアント名" "山口様" "上島様")
for word in "${BANNED_WORDS[@]}"; do
  grep -q "$word" "$DRAFT_FILE" && echo "禁止ワード検出: $word" && exit 1
done

echo "品質チェック通過"
```

## プラットフォーム別のベストプラクティス

### X（Twitter）
- フック（最初の1行）で勝負が決まる
- 1投稿 = 1主張
- リンクは本文に入れない（リーチ低下）
- ハッシュタグは不要（日本語圏では効果薄）
- 引用RTは50-100字の鋭い一言見解が最適

### Facebook（個人ブランド・信頼形成目的）
- 800-1500文字の長文が有効
- ストーリーテリング型（こうだった→こう変わった→こう学んだ）
- 技術用語は平易な言葉に翻訳する
- CTAは「DM/壁打ち/紹介」に寄せる
- Xの短文焼き直しは禁止

### note
- 導入で「読者が得るもの」を明示する
- 章立て・見出しで読みやすくする
- 具体例と数字を入れる
- 無料部分で価値を出して、有料への動機を作る

### Threads
- 感情・共感が刺さる（特に女性ユーザー層）
- 体験談・日常の気づき形式
- 長文も読まれやすい
- Xより柔らかいトーンが合う

---

## 禁止パターン（共通）

- 同じ文章を複数プラットフォームにそのまま流用する
- 「劇的に」「革命的に」「やばい」等の誇張
- ファクト確認なしの数字・実績の記載
- 機密情報（クライアント名・金額）の混入
- AI生成臭が強いテンプレ文（「昨今のXの環境において〜」等）
